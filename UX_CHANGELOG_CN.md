# UX/UI 工作流优化说明

## 目标

本版不改变目录扫描、规则库、请求模板和结果存储的原有安全语义；重点补齐操作路径：让用户能在一个页面确认“会扫什么、是否暂停、失败在哪里、下一步怎么处理”。

## 新增：扫描控制台

主界面新增 **扫描控制台** Tab，包含：

- **扫描边界摘要**：Burp Scope 是否强制、当前自动流量来源、失败重试次数；
- **实时任务状态**：排队、执行、待重试、完成、失败、取消、延迟队列及运行中探针数量；
- **明确的语义提示**：Host、状态码和目录层级只筛选结果视图，不会影响扫描发包边界；
- **暂停 / 恢复扫描**：暂停时任务保留并延迟，恢复后继续调度；
- **失败任务工作台**：仅展示 FAILED、RETRYABLE、CANCELLED 任务，包含来源、Host、URL、尝试次数、最后更新时间、失败原因；
- **逐条重试 / 批量重试**：用户可只重试选中的任务，避免一次性重跑全部任务；
- **复制诊断**：复制任务状态、URL、重试次数与失败原因。无法访问系统剪贴板时会写入 Burp 输出日志；
- **右键菜单状态**：明确显示 IContextMenuFactory 是否注册成功，避免用户误以为菜单一定可用。

## 可靠性页改进

- 调度状态由内部枚举文本改为面向操作的中文摘要；
- 可靠性页增加 Scope / 自动来源 / 右键菜单兼容提示；
- 保留原来的重试、待处理上限、结果分页和索引调试能力。

## 结果页改进

扫描摘要增加固定前缀：

> 结果视图筛选（不影响扫描范围）

因此，Host、状态码和目录层级的作用不再与 Burp Scope、白名单/黑名单混淆。

## 线程模型

- 扫描控制台所有控件、表格与定时刷新均运行在 EDT；
- 调度器传递给界面的数据为不可变 `TaskView` 快照；
- 工作线程仍不直接依赖 Swing；
- 暂停命令由工作台状态与遗留 `scanPaused` 字段共同生效，避免旧界面和新控制台状态不一致。

## 已知边界

- “扫描选中请求 / 扫描当前 Host”仍依赖 Burp 的 `IContextMenuFactory`。某些旧版或裁剪 API 环境不支持该接口时，控制台会明确显示未注册，而不会伪装为可用。
- 该版没有新增一个独立的 URL 输入框来手工创建请求；手工扫描入口仍以 Burp 流量右键菜单为准。
- 任务记录受原补丁会话上限约束，插件卸载后不持久化任务状态。
